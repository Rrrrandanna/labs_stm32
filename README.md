## Быстрый старт 

Этот репозиторий является **основным (upstream)**.
Вы не должны пушить изменения напрямую в него. Работа ведётся через **fork**.
После первой установки GIT выполните следующие команды в терминале git bash:

```bash
git config --global user.email "ваша почта"
git config --global user.name "ваш ник"
```

### 1. Сделайте fork репозитория

1. Откройте страницу репозитория.
2. Нажмите кнопку **Fork** в правом верхнем углу.
3. В вашем аккаунте появится копия репозитория.

---

### 2. Клонируйте свой fork

```bash
git clone https://github.com/YOUR_USERNAME/labs_stm32.git
cd labs_stm32
```

---

### 3. Добавьте оригинальный репозиторий как upstream

```bash
git remote add upstream https://github.com/element3309/labs_stm32.git
```

Проверьте:

```bash
git remote -v
```

---

### 4. Работайте в своей ветке

Создавайте ветку для каждой лабораторной работы:

```bash
git checkout -b lab1-surname        # например: lab1-ivanov
```

Внесите изменения и закоммитьте:

```bash
git add .
git commit -m "lab1 completed"
```

---

### 5. Отправьте изменения в свой fork

```bash
git push origin lab1-surname
```

---

### 6. Создайте Pull Request

1. Откройте ваш fork на GitHub.
2. Нажмите **Compare & Pull Request**.
3. Отправьте Pull Request в основной репозиторий.

---

### 7. Подтягивайте обновления из основного репозитория

Перед началом новой работы обновите локальную копию:

```bash
git checkout master
git pull upstream master
git push origin master
```

---

### **Полезные команды**
```bash
git status          # посмотреть какие файлы изменены
git log --oneline   # история коммитов
git pull            # обновить ветку (перед новой работой)
```


## **Лабораторные работы по STM32 *(распределить по папкам)***

| № | Тема | Ключевые понятия | Что делаем |
|:-:|------|------------------|------------|
| **1** | **GPIO** `💡` | • Цифровые входы/выходы<br>• Тактирование периферии<br>• Режимы работы пинов | Простой blink: настройка ножки микроконтроллера на выход и мигание светодиодом в бесконечном цикле. Знакомство с CubeMX и структурой HAL-проекта. |
| **2** | **TIM** `⏱️` | • Таймеры общего назначения<br>• Предделитель (PSC) и период (ARR)<br>• Прерывания по переполнению<br>• Колбэк-функции (`HAL_TIM_PeriodElapsedCallback`) | Запуск таймера TIM6, мигание светодиодом в обработчике прерывания. Настройка точных временных интервалов. |
| **3** | **Stepper motor** `⚙️` | • Шаговый двигатель<br>• Полношаговый / полушаговый режимы<br>• Конечные автоматы (state machine)<br>• Последовательность коммутации фаз | Управление шаговым двигателем через переключатель (`switch-case`). Пошаговое переключение обмоток для вращения ротора. |
| **4** | **ADC** `📊` | • Аналого-цифровой преобразователь<br>• Опорное напряжение<br>• Разрешение (12 бит)<br>• Инжектированные / регулярные группы | *В разработке.* Измерение аналогового сигнала (потенциометр, датчик). |
| **5** | **UART** `📡` | • Асинхронный последовательный интерфейс<br>• Скорость (бодрейт)<br>• Прерывания при приёме/передаче<br>• Форматирование строк (`printf` через UART) | Передача данных на ПК (терминал) и приём команд. Отправка результатов АЦП или состояния системы. |

---

### **Дополнительные материалы**

| Работа | Полезные макросы / функции HAL | Примечания |
|--------|-------------------------------|------------|
| **GPIO** `💡` | `HAL_GPIO_WritePin()`<br>`HAL_GPIO_TogglePin()`<br>`__HAL_RCC_GPIOx_CLK_ENABLE()` | Настройка пина через `GPIO_InitTypeDef` |
| **TIM** `⏱️` | `HAL_TIM_Base_Start_IT()`<br>`HAL_TIM_PeriodElapsedCallback()` | Расчёт PSC и ARR: `Частота = Тактовая / (PSC+1) / (ARR+1)` |
| **Stepper** `⚙️` | `HAL_GPIO_WritePin()` для каждого вывода обмоток | Обычно 4 вывода обмоток (Ap, An, Bp, Bn) |
| **ADC** `📊` | `HAL_ADC_Start_IT()`<br>`HAL_ADC_ConvCpltCallback()` | Калибровка АЦП и расчет результатов |
| **UART** `📡` | `HAL_UART_Transmit()`<br>`HAL_UART_Receive_IT()` | Перенаправление `printf` через переопределение `_write` |
