# Лабораторная работа: Аналого-цифровое преобразование (АЦП) на STM32

В этом проекте для работы с АЦП подготовлены готовые функции. Вам не нужно разбираться в регистрах или HAL – просто используйте их по инструкции.

## Доступные функции

| Функция | Описание |
|---------|----------|
| `plt_adc_start()` | Запускает однократное измерение АЦП. |
| `plt_adc_conversion_poll()` | Ожидает завершения преобразования (блокирующий опрос). |
| `plt_adc_get_value()` | Возвращает последнее измеренное значение (тип `uint32_t`). |
| `plt_adc_stop()` | Останавливает АЦП (отключает его для экономии энергии). |

> **Примечание:** Все функции уже реализованы на основе HAL. Подключение пинов, тактирование и настройка АЦП выполнены заранее.

## Задание 1
Используя вызовы предложенных выше функций, напишите код так что бы в циклическом режиме выполнялось следующее:
- запуск АЦП
- ожидание преобразования
- получение последнего измеренного значение
- останов АЦП.

 Используя режим отладки (debug) посмотрите результат 
 ```c
 plt_adc_get_value()
```
При превышении порогового уровня в 3000 значений зажгите светодиод на плате.

## Задание 2
Напишите свою функцию 
```c
float plt_adc_get_voltage(void)
``` 
которая позволяет определять напряжение на входе АЦП в вольтах. Добавьте вызов функции в код, проверьте её работу в режиме отладки и реализуйте индикацию светодиодом по превышении на входе значения в `3.0В`.
